# SWC Digital 3.1.0 — Plan 2: Schema v4 + AP Recovery + Pair Flow

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development.

**Goal:** Bump Settings schema to v4 with pairing state (H1 + AP-random-password); implement `POST /api/pair` (open in Setup AP only) that accepts a Crockford-Base32 pairing key, computes H1, and locks the device; turn on `Security::authorize()` so every protected route requires Digest; show the AP recovery password on the physical screen when in Setup mode.

**Architecture:** Settings gets `pairedH1` (32-hex MD5) and uses the existing `apPass` field for a freshly-randomised setup password on first boot of v4 (if empty). `POST /api/pair` is the only route that flips `paired=true` and is reachable only when `netMode()==NET_AP` AND not yet paired; in STA mode it always 404s. `Security::authorize(server, recovery)` becomes `recovery || !paired || verifyDigest()`. Every existing handler (root, status, config GET/POST, scan, export/import, reboot, factory, checkupdate, selfupdate, update, usage GET/POST) gains a one-line `if (!g_security.authorize(server, netMode()==NET_AP)) return;` guard at the top — captive-probe paths stay exempt so the OS popup still fires.

**Tech Stack:** ESP8266WebServer Digest (`requestAuthentication(DIGEST_AUTH)`, `authenticateDigest(user, h1)`), ArduinoJson 7, MD5Builder, LittleFS, Gfx text drawing for the on-screen recovery code.

**Branch:** `feature/v3.1-secure-pairing`.

**Depends on:** Plan 1 complete (Security seam + `/api/identity` + Device ID).

**Spec source:** §"การแก้ระบบ" (security settings schema v4, AP password random 12+, display on screen, OTA auth in UPLOAD_FILE_START), §"Interface และพฤติกรรมใหม่" (`POST /api/pair`, Digest on every other route), §"สมมติฐาน" (forced re-pair migration).

**Spec rule:** "เมื่ออัปเกรดแล้ว AP password ยังว่าง ให้สร้างรหัสสุ่มอย่างน้อย 12 ตัว".

- [ ] **Step 2.1: Add a random-password helper in `Settings.cpp`.**

Near the top of `Settings.cpp`, after the includes:

```cpp
// 12-char alphanumeric (no ambiguous chars like O/0/I/1) for the Setup AP.
// Uses ESP.getChipId() + micros() as entropy; NOT cryptographic, but the
// threat model is "friend on the same Wi-Fi", not a targeted attacker.
static String randomApPass() {
  static const char ALPHABET[] = "ABCDEFGHJKLMNPQRSTUVWXYZabcdefghijkmnopqrstuvwxyz23456789";
  uint32_t seed = ESP.getChipId() ^ micros() ^ (millis() << 8);
  randomSeed(seed);
  String out;
  for (uint8_t i = 0; i < 12; i++) {
    out += (char)ALPHABET[random(sizeof(ALPHABET) - 1)];
  }
  return out;
}
```

```cpp
#include "Platform.h"
#include "config.h"
#include "Net.h"
#include "Settings.h"        // for saveSettings() in pair()
#include <ArduinoJson.h>
#include <ESP8266WebServer.h>   // WebServerClass is a using-alias; need the real type

Security g_security;
const char* SECURITY_REALM = "swc-digital";
const char* SECURITY_USER  = "admin";

void Security::begin() {
  uint32_t id = platformChipId();
  snprintf(deviceId_, sizeof(deviceId_), "%08x", (unsigned)id);
  paired_ = false;
  h1_     = "";
  recoveryOpen_ = false;
}

const char* Security::deviceId() const  { return deviceId_; }
bool Security::paired() const           { return paired_; }
void Security::setPaired(bool p)        { paired_ = p; }
void Security::setH1(const String& h1)  { h1_ = h1; }
const String& Security::h1() const      { return h1_; }
bool Security::hasH1() const            { return h1_.length() == 32; }
void Security::setRecoveryOpen(bool b)  { recoveryOpen_ = b; }
bool Security::recoveryOpen() const     { return recoveryOpen_; }

bool Security::authorize(WebServerClass& server, bool recovery) const {
  (void)recovery;
  // No H1 yet => setup mode, everything is open. The pair endpoint itself
  // enforces "unpaired only" separately (Task 4).
  if (!hasH1()) return true;
  // Paired: require Digest for every protected route. The ESP8266WebServer
  // API issues the 401 challenge itself if authenticateDigest returns false.
  if (!server.authenticateDigest(SECURITY_USER, h1_.c_str())) {
    server.requestAuthentication(DIGEST_AUTH, SECURITY_REALM);
    return false;
  }
  return true;
}

bool Security::pair(const String& h1) {
  if (paired_ || hasH1()) return false;   // already paired -> caller 409s
  if (h1.length() != 32) return false;    // MD5 hex sanity
  h1_     = h1;
  paired_ = true;
  // Persist (the caller passes Settings; we read the global g_settings).
  // The actual persist happens via Settings save in WebPortal handlePair
  // because that's where the Settings& lives. Here we just update state.
  return true;
}

void Security::serializeIdentity(String& out) const {
  JsonDocument doc;
  JsonObject o = doc.to<JsonObject>();
  o["id"]      = deviceId_;
  o["fw"]      = FW_NAME;
  o["version"] = FW_VERSION;
  o["paired"]  = paired_;
  o["mode"]    = (netMode() == NET_AP) ? "ap" : "sta";
  serializeJson(doc, out);
}
```

**Note on `authenticateDigest`:** the ESP8266WebServer signature is `bool authenticateDigest(const String& username, const String& h1)`. The H1 is `MD5(username:realm:password)` lowercase hex — the device stores this hash, never the plaintext password. The Mac computes the same H1 from the pair key (Plan 3).

- [ ] **Step 4.1: Add `handlePair` + a helper to compute H1.**

```cpp
// MD5("admin:<realm>:<pairkey>") lowercase hex. Uses MD5Builder from the
// ESP8266 core. Pure function so a future test can mirror it in Python.
#include <MD5Builder.h>
static String computeH1(const String& pairkey) {
  MD5Builder md5;
  md5.begin();
  String s = String(SECURITY_USER) + ":" + SECURITY_REALM + ":" + pairkey;
  md5.add(s);
  md5.calculate();
  return md5.toString();   // 32 lowercase hex
}

// AP-only, unpaired-only pairing endpoint. The owner reads the AP password
// off the screen, joins the SmallTV-Setup AP, then POSTs the pair key they
// chose (or that the Mac `pair` command generated for them).
static void handlePair() {
  // Hard guard: AP mode + unpaired. In STA mode this route is unreachable
  // (it is registered but always 404s below). Re-check anyway in case of
  // captive-portal weirdness.
  if (netMode() != NET_AP) { server.send(404, "text/plain", "Not found"); return; }
  if (g_security.paired() || g_security.hasH1()) {
    server.send(409, "application/json", "{\"ok\":false,\"error\":\"already paired\"}");
    return;
  }
  if (!server.hasArg("plain")) { server.send(400, "application/json", "{\"ok\":false,\"error\":\"no body\"}"); return; }

  JsonDocument doc;
  if (deserializeJson(doc, server.arg("plain"))) {
    server.send(400, "application/json", "{\"ok\":false,\"error\":\"bad json\"}");
    return;
  }
  // Pair key: 16-char Crockford Base32 (the Mac `pair` command's format).
  // We accept any non-empty printable string ≥12 chars so a human-typed
  // passphrase also works.
  const char* key = doc["pairkey"] | "";
  size_t len = strnlen(key, 256);
  if (len < 12 || len > 128) {
    server.send(400, "application/json", "{\"ok\":false,\"error\":\"pairkey must be 12..128 chars\"}");
    return;
  }
  String h1 = computeH1(String(key));
  if (!g_security.pair(h1)) {
    server.send(409, "application/json", "{\"ok\":false,\"error\":\"pair failed\"}");
    return;
  }
  // Persist into Settings so it survives reboot.
  S->pairedH1 = h1;
  saveSettings(*S);
  server.send(200, "application/json", "{\"ok\":true}");
  // Do NOT reboot: the device stays in AP mode and serves the WebUI with
  // Digest now on, so the same browser session can finish configuration.
}
```

Append to `tests/_security_rules.py`:

```python
import hashlib

REALM = "swc-digital"
USER  = "admin"

def compute_h1(pairkey: str) -> str:
    """Mirror of Security.cpp computeH1(). MD5(user:realm:pairkey) lowercase hex."""
    return hashlib.md5(f"{USER}:{REALM}:{pairkey}".encode()).hexdigest()

# Route-policy matrix: returns the expected HTTP status code for a given
# (device_state, mode, route, has_auth_header) combination. Mirrors Plan 2
# Task 5 logic. Used by test_security.py's RoutePolicyTests.
#
# device_state in {"unpaired", "paired"}
# mode         in {"ap", "sta"}    (ap = Setup AP, sta = joined home Wi-Fi)
# route        in {"identity", "pair", "root", "config_get", "config_post",
#                  "status", "scan", "export", "import", "reboot", "factory",
#                  "checkupdate", "selfupdate", "update", "usage_get",
#                  "usage_post", "captive_probe"}
# has_auth     in {True, False}    (valid Digest header when True)
def route_status(device_state, mode, route, has_auth):
    # Public routes: never auth.
    if route in ("identity", "captive_probe"):
        return 200
    # Pair route: AP only, unpaired only.
    if route == "pair":
        if mode != "ap":                return 404
        if device_state != "unpaired":  return 409
        return 200
    # Everything else:
    if device_state == "unpaired":      return 200   # setup mode, open
    # Paired:
    if has_auth:                        return 200
    return 401
```
